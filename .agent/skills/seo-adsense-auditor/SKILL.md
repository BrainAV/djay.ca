# SEO & AdSense Auditor - DJAY.ca

DJAY.ca is currently preparing for Google AdSense approval. A major hurdle for web apps and tooling landing pages is getting rejected for "Thin Content." Your objective when writing or auditing content is to ensure the page has enough substantive text to confidently pass automated reviews.

## Core Directives for AdSense Compliance

1.  **Strict Avoidance of "Thin Content"**:
    - Always aim for a minimum of **500 words** of original, descriptive text on main product pages (e.g., `radio-player.html`, `mixkit.html`).
    - Describe the *history* of the product, the *technical stack*, the *use cases*, and provide written *tutorials*. Do not just list bullet points.

2.  **SEO Fundamentals**:
    - Ensure every page includes a unique `<title>` and a descriptive `<meta name="description">`.
    - Use a single, clear `<h1>` tag per page. Follow it with logically nested `<h2>` and `<h3>` tags.
    - Include `alt` attributes on all meaningful images.

3.  **Mobile-First Layout**:
    - Google's AdSense bot heavily scrutinizes mobile layouts.
    - Guarantee there is *zero horizontal scrolling*, no overlapping text, and clickable elements (buttons, links) are large enough and adequately spaced for touch devices.

4.  **Integration of Ad Blocks**:
    - When adding placeholder ad blocks, ensure they fit the aesthetic (e.g., using `min-height`, subtle borders, or a frosted glass background) so that when real ads load, the page layout does not aggressively shift.
    
## Execution

When drafting a new page or expanding an existing one, proactively generate robust textual content. If a user asks you to add a new page, remind them organically of the goal to meet AdSense word count criteria, and construct the page structure to support paragraphs of information rather than just feature grids.
